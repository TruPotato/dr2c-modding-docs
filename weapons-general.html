General Weapons Info


Inputs



Variable types are put in brackets, e.g. `custom`[name]` allowing any text input, `power`[number]` requiring a numerical input. This formatting is used in this guide for clarity.
[name] - Any text input using letters, punctuation, numbers, or spaces.
[designation] - Any text input using only letters.
[number] - Any input using a decimal number.
[bool] - An input of either `0` or `1`, representing "false" and "true", respectively.
[range] - An input in a range designated in the command's description.
[angle] - Any input using a decimal number between -180 and 180.
[chance] - Any decimal input between 0 and 1 to denote a percent chance. For instance, a chance value of 0.3 is equal to 30%.
[unique] - Any input described in the command's description.


Weapon Naming



custom: [name] - Create a weapon from scratch. Stats are assigned via tabbed commands under the weapon.
[weapon designation] clone-to: [name] - Create a weapon using stats from a `custom` weapon. Stats assigned to the clone overwrite the pre-existing values given from the `custom` weapon.
weapon: [weapon designation] - Give a weapon a designation. Weapon designations are required to be one word (e.g. the Sniper Rifle's designation is `silentrifle`) and are used in the `clone-to` command.

Appearance and Audio



spriteid: [number] - Choose the sprite for a weapon from the file `dr2c_weapons` under the path `DeathRoadToCanada/gfx/items`. Every cell is 8x32 pixels and the engine assigns a number to each cell, starting with 0 in the top left most cell. For instance, weapons using the Wrench sprite would use `spriteid`17`, as it is the 18th sprite.
DISCLAIMER - There is an alternate sprite designation system that vanilla weapons use, involving hexidecimal numbers and sprite offsets. The previously described method works identically to the vanilla system.
spriteid16x32: [number] - Choose the sprite for a weapon from the file `dr2c_weapons-16x32px` under the path `DeathRoadToCanada/gfx/items`.
